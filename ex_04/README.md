# NumPy 핵심 정리: 고유값/고유벡터 & 배열 형태 처리

## 1. 고유값(Eigenvalue)과 고유벡터(Eigenvector)

### ✔ 함수

```python
w, V = numpy.linalg.eig(A)
```

### ✔ 의미

* `A`: 정방행렬 (n × n)
* `w`: 고유값(Eigenvalues) 벡터 (길이 n)
* `V`: 고유벡터(Eigenvectors) 행렬 (n × n)

### ✔ 관계

각 고유값과 고유벡터는 다음과 같이 대응됨:

* `w[0]` ↔ `V[:, 0]`
* `w[1]` ↔ `V[:, 1]`
* ...
* `w[i]` ↔ `V[:, i]`

즉,
👉 **행렬 V의 i번째 열(column)이 w[i]에 대응하는 고유벡터**
고윳값 벡터의 i 번째 원소와 고유벡터 행렬의 i 번째 열을 곱한게 방정식 Ax = λx를 만족함. 

### ✔ 수학적 정의

각 고유값과 고유벡터는 다음 관계를 만족:

```
A · v = λ · v
```

* `λ`: 고유값 (w[i])
* `v`: 고유벡터 (V[:, i])

### ✔ 특징

* NumPy가 반환하는 고유벡터는 일반적으로 **정규화(normalized)** 되어 있음
  → 즉, **단위벡터 (||v|| = 1)**

## 2. 배열의 형태 정보 확인

### ✔ numpy.ndim

```python
numpy.ndim(A)
```
len(A) 는 행의 개수 (몇줄이냐?)
ndim(A) 는 차원의 수 (몇 차원이냐?)
* 배열의 **차원 수 (axis 개수)** 반환
* 예:

  * 1차원 배열 → 1
  * 2차원 행렬 → 2

---

### ✔ numpy.shape

```python
numpy.shape(A)
```

* 배열의 **형태 (크기)** 반환
* 결과: `(행, 열)` 형태

예:

```python
A.shape  # (3, 4)
```

---

### ✔ numpy.size

```python
numpy.size(A)
```

* 배열의 **전체 원소 개수** 반환

예:

```
3 × 4 배열 → size = 12
```

---

## 3. 배열의 형태 조작 (Reshaping & Flattening)

### ✔ numpy.ravel

```python
numpy.ravel(A)
```

* 배열을 **1차원으로 펼침(flatten)**
* 원소 순서를 유지하면서 나열

예:

```
[[1, 2],
 [3, 4]] → [1, 2, 3, 4]
```

---

### ✔ numpy.reshape

```python
numpy.reshape(A, (m, n))
```

* **데이터 변경 없이 형태만 변경**
* 반드시 **원소 개수 동일해야 함**

예:

```python
A = [1,2,3,4]
A.reshape(2,2)
```

---

### ✔ numpy.resize

```python
numpy.resize(A, (m, n))
```

* 배열 크기를 변경하면서 **원소 수가 달라도 허용**
* 특징:

  * 원소가 부족하면 → **반복**
  * 원소가 많으면 → **잘림**

예:

```python
A = [1,2,3]
numpy.resize(A, (2,2))
```

결과:

```
[[1,2],
 [3,1]]   # 반복됨
```

---

## 🔥 핵심 차이 요약

| 함수      | 특징                | 원소 개수 유지  |
| ------- | ----------------- | --------- |
| ravel   | 1차원으로 펼침          | 유지        |
| reshape | 형태만 변경            | 유지 (필수)   |
| resize  | 크기 변경 + 데이터 변경 가능 | 유지 안 해도 됨 |

---

## ✔ 한 줄 요약

* `eig`: 고유값 ↔ 고유벡터는 **열 기준으로 매칭**
* `shape / ndim / size`: 배열 구조 파악
* `reshape`: 안전한 형태 변경
* `resize`: 위험하지만 유연한 크기 변경
